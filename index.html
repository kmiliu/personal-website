VTA–PFC Neural Coordination

Problem: How does ethanol alter VTA–PFC neural coordination during social interaction?

Approach: Built end-to-end pipelines for PSD, coherence, PAC, and Granger causality on LFP signals.

Result: Discovered ethanol-induced changes in theta–gamma coupling and cross-region interactions.

Your Role

  • Built end-to-end analysis pipelines from raw LFP to publication-ready figures
  • Integrated MATLAB + Brainstorm + Python workflows for seamless processing
  • Automated event-aligned processing for behavioral epochs
  • Implemented PSD, coherence, PAC, and Granger causality analyses
  • Discovered ethanol-specific effects on theta–gamma coupling during social approach

Methods & Tools

  • Signal processing: Bandpass filtering, Hilbert transform, time-frequency decomposition
  • Metrics: Power spectral density (PSD), coherence, phase–amplitude coupling (PAC), Granger causality
  • Tools: MATLAB, Brainstorm toolbox, Python (NumPy, SciPy), data visualization
  • Workflow: Event-aligned epoching, automated batch processing, reproducible pipelines

Key Findings

Integrated behavioral segmentation with neural data to examine directional connectivity and individual variability in affect-modulated circuit communication. Presented findings at Summer Symposium 2025 and Research Pathway 2026 to over 200 students.

Reproducibility

  • Data: Internal lab dataset (Melón Lab, Wesleyan University)
  • Code: Available upon request / private GitHub
  • Environment: MATLAB R2023b, Python 3.10, NumPy, SciPy

Next Steps & Reflection

This project taught me the value of building flexible, end-to-end pipelines that can handle complex experimental designs. The integration of multiple analysis methods (PSD, coherence, PAC, Granger causality) revealed patterns that no single metric could capture alone. I'm particularly interested in extending this work to understand how individual differences in circuit dynamics relate to behavioral variability.

Boolean Genetic Networks

Problem: How sensitive are genetic regulatory networks to mutations across different attractor states?

Approach: Simulated Kauffman Boolean networks with mutation sensitivity analysis optimized for HPC.

Result: Revealed attractor-dependent vulnerability patterns and optimized computational performance for large-scale simulations.

Your Role

  • Designed and implemented Boolean network simulations (Kauffman nets)
  • Developed mutation sensitivity analysis across different attractor states
  • Optimized memory usage and computational performance for HPC clusters
  • Parallelized simulations using job arrays and efficient resource allocation

Methods & Tools

  • Model: Boolean genetic regulatory networks (Kauffman networks)
  • Analysis: Attractor identification, mutation sensitivity, robustness measures
  • Optimization: Memory-efficient data structures, parallel processing, HPC job scheduling
  • Tools: Python, NumPy, HPC (Slurm), computational biology modeling

Key Findings

Different attractor states showed varying degrees of sensitivity to mutations, suggesting that network topology and current state jointly determine robustness. Some attractors were remarkably stable, while others were highly vulnerable to single-node perturbations. This has implications for understanding biological robustness and evolvability.

Reproducibility

  • Code: Python simulation framework with configurable network parameters
  • HPC: Slurm job scripts, resource allocation strategies
  • Environment: Python 3.10, NumPy, HPC cluster environment

Next Steps & Reflection

This project taught me the importance of computational efficiency in large-scale simulations. What started as a straightforward modeling question became an exercise in optimization and parallel computing. I'm interested in extending this approach to more realistic network topologies informed by empirical gene regulatory data, and exploring how network structure shapes evolutionary dynamics.

Building Evidence Pipelines

My technical strength lies in establishing end-to-end workflows that transform disorganized data into reproducible evidence with clear visualization. In neuroscience projects, I routinely process LFP and behavioral data from VTA–PFC circuits.

Challenge → Action → Result: Raw VTA–PFC LFP data arrived with inconsistent epochs and mixed behavioral conditions. I bridged Brainstorm and MATLAB—performing frequency suppression, bandpass filtering, event-based epoch segmentation (social vs. non-social, EtOH vs. saline), and baseline correction—then conducted coherence and Granger causality analyses, distinguishing intra-area (iPAC) and inter-area (irPAC) phase-amplitude coupling. Result: Reproducible pipelines that translate statistical analyses into charts and conclusions supporting evidence-based arguments.

Machine learning pipeline: Using PyTorch, I implement MLP prototypes to understand trade-offs between width, depth, and expressiveness. For text scoring interpretability, I combine HuggingFace and SHAP to generate token importance distributions with confidence intervals, pinpointing elements influencing model decisions and screening for potential biases.

I continuously refactor fragmented scripts into parameterized, modular functions with unit tests and small-sample regression checks. Combined with Git/GitHub version control, documented README files, and maintained runtime environments, I strive for reproducibility: same raw data + same parameters = same results.
